'@forinda/kickjs-db': minor
---

Generated migration SQL files now carry an immutable provenance banner (`-- Generated by @forinda/kickjs-db vX.Y.Z — review state lives in meta.json`) instead of the mutable `-- REVIEWED: false` marker. Review state lives ONLY in `meta.json` — which the integrity hash does not cover — so reviewing a migration (or hand-flipping anything cosmetic) can no longer invalidate the journal hash. `kick db migrate review <id>` now flips `meta.json` only for banner-era migrations; legacy migrations with in-file markers still get the swap-and-rehash treatment. The unreviewed-migration error now names the review command instead of telling you to edit the marker.
